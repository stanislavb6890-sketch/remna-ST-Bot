Сравнение API Remna (api-1.yaml) и кода панели — действия со сквадами
========================================================================

В API Remna есть массовое действие со сквадами:
----------------------------------------------------------------------
  POST /api/users/bulk/update-squads
  Тело: { uuids: string[], activeInternalSquads: string[] }
  Описание: разом выставить activeInternalSquads для до 500 пользователей.

В панели это не используется:
----------------------------------------------------------------------
  - В remna.client.ts функция remnaBulkUpdateUsersSquads() объявлена (для
    совместимости со спекой API), но ни один роут и ни один сервис панели
    её не вызывает.
  - Импорт remnaBulkUpdateUsersSquads в admin.routes.ts удалён, чтобы не
    создавать впечатление, что в админке есть массовое изменение сквадов.

Что панель реально использует:
----------------------------------------------------------------------
  1) PATCH /api/users (один пользователь)
     - Тело: uuid, activeInternalSquads, expireAt, telegramId, email и т.д.
     - Вызовы: syncToRemna, админка (редактирование клиента в Remna), триал,
       промо-группа, промокод FREE_DAYS, активация тарифа.

  2) POST /api/internal-squads/{squadUuid}/bulk-actions/add-users
     - Тело: { userUuids: string[] }
     - Вызовы: после установки сквада при триале, промо-группе, промокоде
       FREE_DAYS и активации тарифа (передаём по одному userUuid, не массу).

  3) DELETE /api/internal-squads/{squadUuid}/bulk-actions/remove-users
     - Тело: { userUuids: string[] }
     - Вызовы: только в админке «Удалить из сквада» по одному клиенту
       (POST /admin/clients/:id/remna/squads/remove).

Итог:
----------------------------------------------------------------------
  Массовых действий со сквадами в панели нет. Нет ни одного вызова
  POST /api/users/bulk/update-squads, поэтому с панели невозможно
  случайно выставить одни и те же сквады сразу многим пользователям
  или «обнулить» сквады массово.
